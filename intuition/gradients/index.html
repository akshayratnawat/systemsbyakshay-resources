Concept 01 of 06

What Is a Gradient

A gradient tells you which way is uphill and how steep the hill is at your current point.
The intuition

For a loss function L(w), the quantity ∂L/∂w is the local slope at the current weight value. If the slope is positive, nudging w upward makes loss go up. If the slope is negative, nudging w upward makes loss go down.

That is why training does not move in the direction of the gradient. It moves in the opposite direction. The gradient points uphill. Optimization wants downhill.

Diagram - reading the slope
Current point
→
Gradient > 0
Loss goes up if w increases
Current point
→
Gradient < 0
Loss goes down if w increases
To minimize loss, step opposite the sign
You can think of the gradient as a tiny arrow attached to the current point on the loss landscape.
Fast lookup
Gradient signWhat increasing w does
∂L/∂w > 0Loss increases, so w is hurting.
∂L/∂w < 0Loss decreases, so w is helping.
∂L/∂w = 0Flat point locally. No immediate slope signal.
Key insight: A gradient is not the loss value. It is the sensitivity of the loss to a tiny change in a weight.
Concept 02 of 06

Partial Derivatives

A modern network has too many weights for one slope. You need one gradient per parameter.
The intuition

A partial derivative ∂L/∂wi asks one tightly scoped question: if you change only wi and freeze every other weight, how does the loss respond?

That framing matters because neural networks are huge coupled systems. The only practical way to update them is to measure the slope for every parameter separately, then bundle those slopes into a full update.

Diagram - one weight at a time
Change w1
Freeze w2 ... wn
Measure ∂L/∂w1
Repeat for every weight
One training step = one slope per parameter
Backpropagation is the engine that computes all those partials efficiently in one backward pass.
Scale check
Model sizeGradients per step

What the notation means
TermMeaningWhy it matters
LThe loss you are trying to minimize.The objective for training.

Concept 03 of 06

The Gradient Vector

Stack every partial derivative together and you get the direction of steepest ascent in weight space.
The intuition

Once you have a partial for every parameter, you pack them into one object: the gradient vector ∇L. That vector points in the direction that makes loss increase fastest.

If w1 has gradient +0.42, then increasing it makes the loss worse. If w2 has gradient -0.18, then increasing it helps. Gradient descent flips the whole vector and walks the other way.

Diagram - stacking partials
∂L/∂w1
∂L/∂w2
∂L/∂w3
...
∂L/∂wn
Together they form one direction in parameter space
The more parameters you have, the higher-dimensional this direction becomes. The intuition stays the same.
Reading components
WeightGradientInterpretation
w1+0.42Making loss worse if increased.
w2-0.18Helping loss if increased.
w30.00Little local effect right now.
The formula
∇L = [∂L/∂w1, ∂L/∂w2, ..., ∂L/∂wn]
The gradient vector is every partial derivative stacked into one object.
  • ∇L - full gradient across all parameters.
  • ∂L/∂wi - slope contributed by one parameter.
  • Direction - points toward steepest ascent, so descent moves opposite.
Concept 04 of 06

Gradient Descent

Training is a loop: compute loss, compute gradients, update weights, repeat.
The intuition

Gradient descent is the mechanical rule that turns slope information into learning. The update says: move each weight opposite its gradient, scaled by the learning rate η.

If the gradient is large, the weight wants a larger correction. If the learning rate is large, the optimizer takes a bigger step. The same logic applies whether you train a tiny regression model or a frontier-scale transformer.

Diagram - the loop
1. Compute loss
2. Compute gradients with backprop
3. Update weights
4. Repeat for the next batch
This four-step loop is the heartbeat of training.
Four-step breakdown
StepWhat happens
1Run the model and compute current loss.
2Backprop computes gradients for every weight.
3Optimizer updates weights using gradients and η.
4Process the next batch and repeat.
The update rule
w ← w - η · ∂L/∂w
Subtract the gradient because the gradient points uphill.
  • w - the current weight value.
  • η - learning rate, or step size.
  • ∂L/∂w - slope telling you how loss changes near the current point.
Key insight: Basic gradient descent has one crucial hyperparameter: the learning rate. Everything else is the mechanics of measuring and applying slope.

Concept 05 of 06

Learning Rate

The learning rate decides how big each downhill step should be.
The intuition

If η is too large, updates overshoot the valley and loss can explode. If η is too small, training still works in principle but takes far longer than necessary.

That is why learning rate tuning is usually the first lever to pull when training looks unstable or painfully slow.

Diagram - step size regimes
1e-1
Too large - overshoots and diverges
3e-4
Sweet spot for many transformer runs
1e-6
Too small - training drags
Same optimizer, same model, completely different behavior if the step size is wrong.
Practical tuning table
SituationAction
Training explodesHalve the learning rate and retry.
Training is stable but slowTry about 3x larger and watch loss.
Long runsAdd a scheduler so the rate can decay over time.
LR finder workflow
StepWhat to doWhy
1Start very low, around 1e-7.Avoid immediate divergence.
2Increase LR exponentially every batch.Cover many scales in one sweep.
3Track where loss starts rising sharply.That is the danger zone.
4Pick a value just before the rise.Usually gives a fast but stable starting point.
Warning: A bad learning rate can look like a broken architecture, a bad dataset, or a bug in backprop. Check step size before chasing exotic explanations.
Concept 06 of 06

Vanishing Gradients

When gradients get multiplied by small numbers across many layers, early layers stop learning.
The intuition

Backprop pushes gradients backward through the network by repeated multiplication. If each layer contributes a factor smaller than 1, the signal shrinks fast.

With sigmoid activations, the maximum derivative is only 0.25. Multiply that through enough layers and the earliest weights see almost no update signal. Training stalls even though later layers still receive gradients.

Diagram - shrinking backward signal
L5: 1.00
→
L4: 0.25
→
L3: 0.06
L2: 0.015
→
L1: ≈0
By the first layers, the update can be effectively gone
Vanishing gradients are not abstract math pain. They directly stop early layers from adapting.
Common fixes
FixWhy it helps
ReLUIts max derivative is 1.0, so it preserves signal better.
Residual connectionsCreate shorter gradient paths through deep stacks.
Batch normKeeps activations in healthier ranges for optimization.
Gradient decay by layer
LayerGradient valueWhat it means
L51.00Healthy update signal at the output side.
L40.25Already shrunk by 4x.
L30.06Now very small.
L20.015Barely enough to change weights.
L1≈ 0Almost no learning signal left.
Warning: If your activation derivatives are consistently below 1, depth alone can kill learning. Architecture choices like ReLU and residual paths are optimization tools, not just style preferences.
